# Prompt: 02-demo-deployment-gap-v5-infographic-editorial-set

## Role

Body image for the demo-to-deployment gap section.

## Visual Mode

Operator Diagram

## Visual Weight

medium

## Style Family

Infographic Editorial

## Positive Prompt

Create a premium editorial infographic explaining the difference between a model demo and a trusted deployment.

Core visual predicate: a demo proves capability; deployment proves the organization can operate it.

Composition:
- 16:9 landscape.
- Warm white background, clean editorial spacing.
- Left side: a simple demo card with a bright model spark and a small proof mark.
- Right side: a larger operating loop made of four simple modules: context, permissions, review, outcome.
- A thin line crosses a visible gap from the demo card into the operating loop.
- Use one coral warning marker in the gap, suggesting friction and risk.
- Keep the graphic airy, legible, and calm.
- Use the same visual language as the selected ACTOR infographic probe: modular cards, thin outlines, muted cyan / amber / green accents.

Text guidance:
- Only large readable labels if needed: DEMO, GAP, DEPLOYMENT, CONTEXT, PERMISSION, REVIEW, OUTCOME.
- No tiny labels or paragraphs.

## Negative Prompt

No split screen filled with text, no dashboard, no fake app UI, no robot, no long arrows crossing everywhere, no stock people, no dark room, no dense process map.

## Mode Constraints

One predicate only: demo capability must cross the gap into operating loop.

## Density Budget

Light diagram, one gap, one loop, no more than five labeled elements.

## Prior Failure To Avoid

Avoid repeated glowing glass route language; keep it graphic and editorial.
