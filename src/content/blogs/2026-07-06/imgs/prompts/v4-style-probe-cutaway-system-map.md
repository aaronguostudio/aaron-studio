# Prompt: v4-style-probe-cutaway-system-map

## Role

Style probe for the FDE article. Test whether deployment-native engineering can be shown as a layered operating model.

## Visual Mode

Operator Diagram

## Visual Weight

medium

## Style Family

Cutaway System Map

## Positive Prompt

Create a clean architectural cutaway system map for enterprise AI deployment.

Core visual predicate: FDE connects the model layer to real work through data, permissions, workflow, evaluation, and feedback.

Composition:
- 16:9 landscape.
- A simplified cutaway of an enterprise workflow as stacked layers, like an architectural section drawing.
- Bottom layer: messy existing systems and data sources, represented by simple paper/file/database blocks.
- Middle layer: permissions, guardrails, and integration paths, represented by gates and connectors.
- Top layer: users and business workflow loop, represented by a clean path and outcome marker.
- A forward deployed engineer is shown as a small neutral operator figure working across layers, not as a hero.
- Use precise linework, muted colors, warm paper background, cyan connectors, amber action path, green feedback loop.
- Premium editorial technical illustration, not isometric SaaS art.

Text guidance:
- Avoid text where possible.
- If needed, use only four large labels: DATA, CONTROL, WORKFLOW, FEEDBACK.

## Negative Prompt

No busy isometric city, no server room, no cyberpunk, no tiny labels, no fake code, no dashboard panels, no dense architecture diagram, no logos.

## Mode Constraints

One mechanism only: layers that connect model capability to operating workflow.

## Density Budget

Mechanism diagram, but keep it sparse: three layers, one operator figure, one feedback loop.

## Prior Failure To Avoid

Do not recreate the earlier overloaded platform-convergence image.
