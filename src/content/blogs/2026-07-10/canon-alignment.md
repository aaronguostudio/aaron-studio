# Canon Alignment

## 文章当前判断

文章认为 ChatGPT Work 与 Claude Cowork 的竞争已经从模型层上移到工作控制层。当前证据下，ChatGPT Work 的优势是更广的产品表面积，Claude Cowork 的优势是更多文件、插件与 Office 工作流的实际证据和失败经验。金融团队的选择标准应是四个平面：context、execution、artifact、control。

## 与旧文章的呼应

1. **2026-02-26 金融插件文章**：延续“正确结构的起点比完美答案更值钱”和“skills 是机构知识载体”。新文章把竞争层升级到承载 skills 与交付物的 workspace。
2. **2026-06-20 AI operating system**：延续“任务的单位从 prompt 变成 run”。新文章解释 vendor 正在把 run 做成面向知识工作者的产品。
3. **2026-07-01 The One-Person Project**：延续“AI 增大执行半径，但责任、边界和证据不能外包”。新文章把这一判断应用到金融 agent 选型。
4. **2026-07-06 Deployment Company**：延续“模型能力只有进入可运行、可控制、客户拥有的 workflow 才产生价值”。

## 观点升级

- 旧判断：skills 正在成为产品。
- 新判断：skills 仍是机构知识资产，但 workspace 正在成为竞争性的 control plane；它决定 skills 能读取什么、在哪里运行、产生什么交付物、怎样被审批和恢复。
- 旧判断：AI operating system 需要 intent → run → review → memory。
- 新判断：商业工作台把这个循环拆成 context、execution、artifact、control 四个可采购、可治理的产品平面。
- 旧判断：客户要拥有 workflow。
- 新判断：即使采用多 vendor 栈，组织也要拥有跨产品的 skills、evals、permission rules、audit history 和 runbooks。

## 需要避免的惯性

- 不要再次写成“junior analyst 是否会被替代”。这是旧文章已经避开的低价值叙事。
- 不要把所有文章都硬塞回 ACTOR。四平面已经是本篇唯一的执行框架。
- 不要因为旧文对 Claude 有直接实测，就让新文变成 Claude 胜利宣言。
- 不要把 customer ownership 写成抽象价值观；要落到可迁移 skills、评估集、日志、权限和恢复。
- 不要自我引用超过读者需要。每个站内链接必须帮助理解机制。

## 可以加入的 Aaron 判断

1. "ChatGPT Work has more surface area. Claude Cowork has more scar tissue." 这是当前最清晰、同时承认样本不对称的判断。
2. "For finance, uncertainty surfaced is more valuable than confidence performed." 收据 demo 的 VERIFY rows 可以支撑这个判断。
3. "The workspace is becoming the product, but the workflow still has to belong to the customer." 这是对旧 canon 的升级句。
4. Aaron 的直接经验应限定为：他跑过 Claude finance workflows，知道 editable starting point 的价值；尚未做公平的 ChatGPT Work 长期实测。
