# Canon Note

## Canonical Idea

The competitive layer above models and skills is the work control plane. An AI workspace creates durable value when it can carry institutional context through execution into editable artifacts while preserving permissions, evidence, review, and recovery.

Reader-facing expression: ChatGPT Work has more surface area; Claude Cowork has more scar tissue.

Finance acceptance rule: if the source cannot be traced, the exception cannot be seen, and the workbook cannot be reviewed, the task is not done.

## Reusable Frame

### The Four Planes of an AI Workspace

1. **Context** - sources, rules, templates, prior decisions, institutional knowledge, portability.
2. **Execution** - cloud/local runtime, connectors, browser/computer use, schedules, decision stops.
3. **Artifact** - editable formulas, linked cells, text boxes, citations, exceptions, deliverable quality.
4. **Control** - permissions, approval, lineage, logs, cost, recovery, workflow ownership.

Recommended evaluation pattern: run the same bounded workflow on the same approved or synthetic inputs, then inject a stale source or mapping error and test the return path.

## Claims Added

- The workspace around skills is becoming the product and the strategic control layer.
- Product maturity should be evaluated through visible work and recovery evidence, not feature count alone.
- In finance, surfaced uncertainty can be more valuable than performed confidence.
- A hybrid agent stack is an institutional boundary design problem, not merely a user preference.

## Claims Updated

- "Skills are becoming the product" remains true but incomplete; skills compound only when a workspace can govern how they receive context, act, produce artifacts, and return evidence.
- "The unit of work is a run" now has a vendor-selection consequence: teams must decide who owns the state and controls around the run.
- "The customer must own the workflow" now includes portable skills, templates, evaluation sets, permission rules, audit history, and runbooks across agent vendors.
