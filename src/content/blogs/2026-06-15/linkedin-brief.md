Fable 5 made something clear to me:

The unit of AI work is changing from response to run.

The old AI loop was prompt-response. I stayed inside the work: ask, inspect, correct, add context, ask again.

With a long-running agent, the shape is different. You give it an objective, it plans, uses tools, coordinates work, and comes back with artifacts. The output matters, but the run itself matters more.

That changes the scarce skill.

It is no longer only about writing better prompts. It is about designing the operating contract around the run:

- objective
- context
- authority
- checkpoints
- evidence
- budget
- rollback

This is also why Superpowers clicked for me. It is not another model. It is a skills framework and software development methodology that wraps coding agents in a more disciplined workflow: clarify, spec, plan, execute, review, verify.

The model is the engine. The workflow is the operating discipline around the engine.

My current view: as agents become more autonomous, the advantage will shift from prompting to run design. The best operators will be the people who can let execution run ahead without letting judgment fall behind.
