# Illustration Plan: Personal AI Operating System Atlas

## Visual System

This outline follows `imgs/style.md`. Do not generate from this outline without preserving the style brief's palette, motifs, anti-patterns, and quality checklist.

This post should not use generic "glowing AI brain" imagery. The image set should feel like a coherent data-rich personal operating system, built from Aaron's actual workflow.

**Style name:** Personal AI Operating System Atlas

**Core look:**
- Cinematic editorial base, not corporate SaaS illustration.
- Real-world scenes grounded in travel, laptop/phone work, dashboards, local machines, cloud tasks, and content production.
- Translucent data overlays that feel like a living operating system: routes, task graphs, process lanes, review gates, memory traces, metric signals.
- High contrast but not dark-blue-only. Graphite black, deep green, off-white, electric cyan, warm amber, occasional soft red.
- Fine-line vector overlays, small data nodes, and subtle topographic grids.
- Minimal or no generated text inside body images. Use symbols, panels, graphs, routes, and icons instead of unreliable labels.
- Every image should share the same visual grammar: 16:9, cinematic depth, thin UI lines, luminous data threads, restrained color accents, no decorative blobs.

**Series motifs:**
- A thin amber road line = real life moving forward.
- Cyan task routes = delegated work.
- Green review gates = human judgment checkpoints.
- White memory nodes = repo/docs/metrics/session artifacts.
- Soft red pulse = risk, permissions, or boundary.

**Quality bar:**
- No stock-photo vibe.
- No random dashboards.
- No generic humanoid robot.
- No fake unreadable interface text.
- No cluttered walls of UI.
- Every image must clarify one section's mechanism.
