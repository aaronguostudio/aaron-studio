# Idea: I Gave Codex a Task From a Moving Tesla

## Raw Opening Seed

最近两个月，我用 Claude Code 以及 Codex 彻底改变了我的工作方式。

故事开头：

特斯拉自动驾驶载着我们一家前往加州的路上。我对着 Codex 的 mobile app 说：去 Notion task tracker 里面完成我的 “博客分析 task”，用 Superpowers skill 执行。

就在同时，Claude Code 刚刚完成了我最新的开发任务，总共耗时 1.5 小时。

AI agent 把任务路由到我家里的 AI 工作站或者云端。他们有时候会自动激活好几个 sub agents，有时候会 search 几十个网站或者 API 来更高效地完成工作。

这不是未来，这是现在。

## Editorial Take

这个开头很强，因为它不是抽象地说 “AI 很厉害”，而是把读者直接放进一个已经发生的生活场景：

- 人在移动中，不在办公桌前。
- 工作不是一个 prompt，而是一个委派出去的任务。
- agent 不是只生成答案，而是进入任务系统、知识系统、开发环境和云端/本地算力。
- 同一时间有多个 AI 工作流在后台推进。

最值得写的不是“我用了 Claude Code 和 Codex 变高效了”，而是：

> 过去两个月，我的工作单位从一次 prompt 变成了一个可委派、可审计、会学习的 AI work system。

这篇文章要避免写成工具清单。工具可以出现，但它们应该服务于一个更大的变化：个人操作系统正在被 AI agent 重写。

## Working Thesis

AI 最重要的生产力变化，不是让我把每个任务做快一点，而是让我重新设计了工作本身。

过去，我坐在电脑前，把任务一件一件完成。现在，我更像是在经营一个小型的个人 AI organization：我定义目标、设置约束、选择 skill、分配任务、检查结果、把反馈写回系统，让下一次执行更好。

这不是简单的自动化。自动化执行固定流程；AI agent 执行的是带不确定性的工作。它可以搜索、判断、调用 API、写代码、生成内容、制作图片和视频、操作电脑、安排后台任务，并在失败时尝试新的路径。

## Recommended Angle

**Title direction:** "I Gave Codex a Task From a Moving Tesla"

**Chinese title direction:** "在去加州的特斯拉上，我把工作交给了 Codex"

**Sharper alternative:** "I Stopped Using AI as a Tool. I Started Managing It as a Work System."

**Chinese alternative:** "我不再把 AI 当工具用，而是把它当工作系统管理"

The article should be personal, but operator-grade. It should feel like a field report from someone already living in an AI-native workflow, not a prediction about the future.

## Suggested Structure

### 1. This Is Not The Future

Open with the Tesla / Codex mobile / Notion task tracker / Superpowers scene.

Then zoom out: while one agent is executing a blog analytics task, another agent has just finished a 1.5-hour development task. Some work runs on the home AI workstation, some in the cloud. Agents may call subagents, search dozens of sources, hit APIs, edit files, and return completed work.

Core point: the interface changed from "sit at desk and do work" to "declare intent and manage execution."

### 2. I Started With Coding, Because Coding Was The Obvious Door

Describe the early phase:

- Claude Code and Codex as coding partners.
- Generate specs, plans, tests, implementation, reviews.
- Let agents run for 30-90 minutes instead of asking for tiny snippets.
- The human role moves from typing code to defining the shape of the work and judging the output.

Tie this to the previous Fable / autonomy article without repeating it: coding was where the autonomy became obvious first.

### 3. Then AI Escaped The IDE

The bigger shift happened when the same pattern moved into non-coding work:

- Writing blogs and social posts.
- Generating images and videos, including Seedance experiments.
- Managing computer tasks and administrative work.
- Reading metrics and maintaining a blog growth feedback loop.
- Using mobile voice commands to start work when away from the desk.
- Running scheduled jobs and background workflows.

This section should include concrete examples, not a long list. Pick 4-5 mini-stories.

### 4. The New Workflow: Intent, Skills, Agents, Review, Memory

Introduce a simple operating model:

1. Intent: I define the outcome, not every step.
2. Skill: I route the task into a repeatable workflow.
3. Agents: the system decomposes, searches, calls APIs, writes, or edits.
4. Review: I inspect outputs through quality gates.
5. Memory: the result and feedback go back into the system.

This is where the updated blog skills and self-enhancement mechanism belong.

The data layer is a concrete example: blog posts, distribution channels, Rybbit metrics, YouTube data, LinkedIn gaps, reward scores, postmortems, and weekly reviews can become feedback for what topics, hooks, visuals, videos, and distribution patterns work.

Important framing: this is not full reinforcement learning yet. It is a practical feedback loop that makes the human-agent system smarter over time.

### 5. What Actually Changed In My Day

Make the benefits tangible:

- I can start meaningful work from my phone.
- I can run multiple work streams in parallel.
- I can leave agents to work while I drive, exercise, spend time with family, or read.
- I can ask for research that would have taken me hours.
- I can turn ideas into article, images, video, X post, newsletter, and publishing artifacts.
- I can make administrative tasks less sticky.

But include the limitation:

- The human is still accountable.
- Bad delegation creates bad output faster.
- Review gates, permissions, cost controls, and taste matter more as agents get stronger.

### 6. The Point Is Not To Work Forever

Close with the human angle:

AI should not only make us do more work. It should give back time for things that make life larger: social connection, exercise, reading, travel, family, and deeper thinking.

The optimistic thesis:

> AI will not make humans obsolete. It will make more humans capable of operating at a higher level, if we learn to manage it with judgment.

The world still has too many unsolved problems. AI gives individuals and small teams more leverage to work on them.

## Concrete Examples To Use

- Mobile operation: speaking to Codex mobile while in the Tesla, asking it to complete a Notion task tracker item.
- Long-running dev task: Claude Code completing a development task in about 1.5 hours.
- Scheduled jobs: recurring or background blog growth ingestion/review workflows.
- Automation: routing tasks to home AI workstation or cloud execution.
- Subagents: agents spinning up multiple specialists for research, implementation, verification, or content.
- Research: searching dozens of websites, APIs, docs, or data sources before producing a result.
- Content production: blog workflow generating article, Chinese version, X post, newsletter, image plan, cover, video script, and video.
- Media generation: Seedance video experiments as an example of AI entering creative production.
- Self-learning: blog-growth data model, Rybbit metrics, reward score, postmortems, and weekly feedback loop.

## Risks To Avoid

- Do not sound like an AI influencer listing shiny tools.
- Do not overclaim autonomy: agents still need constraints, verification, and human judgment.
- Do not make the piece only about coding; the point is work-system redesign.
- Do not make it too soft or philosophical. Keep the operator perspective.
- Do not repeat the 2026-06-15 Fable article too closely. This one should be about daily life and personal operating system transformation.
