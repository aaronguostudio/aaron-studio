# Blog Plan: I Gave Codex a Task From a Moving Tesla

## Meta

- **Author:** Aaron Guo
- **Target:** aaronguo.com blog (EN + ZH bilingual)
- **Tone:** Personal, operator-grade, crisp, commercially useful; optimistic without AI-influencer hype.
- **Length:** 1,900-2,400 words for English; Chinese adaptation slightly tighter.
- **Audience:** Builders, product leaders, AI-native operators, and ambitious professionals trying to move beyond casual AI usage.
- **CTA:** newsletter
- **Slug:** `ai-became-my-operating-system`
- **Category:** `personal-operating-system`
- **Tags:** `ai-agents`, `codex`, `claude-code`, `ai-workflow`

## Hook

The opening scene is a family road trip: a Tesla driving toward California, Aaron speaking into Codex mobile and asking it to work through a Notion task tracker item with Superpowers. In parallel, Claude Code has just finished a 1.5-hour development task. The scene should feel specific, almost mundane, and therefore more startling: this is not a staged future-of-work demo; this is an ordinary day.

## Thesis

AI's biggest productivity change is not that it makes individual tasks faster; it changes the unit of work from "I do a task" to "I design a work system that can execute, report back, and learn."

## Personal Anchor

Over the last two months, Aaron moved from using Claude Code and Codex primarily for programming to using agents across writing, blog production, media generation, Notion task routing, scheduled jobs, analytics, and self-improvement loops. The proof points are concrete: an Excel add-in UI consolidation task, Codex mobile controlling work away from the desk, Notion task tracker as a dispatch layer, blog-growth metrics and reward score, and Seedance video lab runs with artifacts, critiques, and lessons.

## Outline

### Part 1: This Is Not A Demo

- Point: The future-of-work moment already arrived inside an ordinary family trip.
- Evidence/example: Tesla to California; Codex mobile; Notion task tracker; Superpowers; Claude Code finishing a 1.5-hour dev task.
- Personal beat: Aaron is not at the desk, but the work is still moving.
- Mechanism: mobile becomes the control surface; agents/workstations/cloud become the execution layer.

### Part 2: Coding Was The First Door

- Point: Coding made agentic work obvious because code has specs, tests, files, diffs, and reviewable artifacts.
- Evidence/example: Excel UI consolidation task: design tokens, chat panel, first-run state, settings, login, ribbon, assets, benchmark gate.
- Personal beat: Instead of asking for snippets, Aaron delegates larger runs and reviews outputs.
- Mechanism: AI moves the human from keystroke execution to outcome definition and quality control.

### Part 3: Then The Pattern Escaped The IDE

- Point: The real change happened when the same loop spread into writing, video, analytics, and admin.
- Evidence/example: blog-production workflow produces article, Chinese version, X post, newsletter teaser, YouTube script, metadata.
- Evidence/example: Seedance lab went from a 4-second smoke test to 15-second storyboard-grid videos and a one-minute prototype, with run records and critiques.
- Evidence/example: scheduled jobs and persistent Mac mini agent are the earlier foundation.
- Personal beat: Work no longer lives inside one app; it moves through a system.

### Part 4: The Operating Model

- Point: The new workflow is not "ask AI for output"; it is intent -> skill -> agent run -> review -> memory.
- Evidence/example: Superpowers standardizes the work process; blog skills encode publishing gates; video lab records prompt, output, critique, and next variation.
- Personal beat: Notion is useful, but the real memory lives across repo docs, metrics, generated artifacts, and sessions.
- Mechanism: skills turn judgment into repeatable process; review gates keep autonomy bounded.

### Part 5: Self-Enhancement Is The Next Layer

- Point: The most interesting systems do not only execute; they learn from what happened.
- Evidence/example: blog-growth loop: publish, distribute, ingest metrics, calculate reward score, run postmortems, feed lessons back into content workflows.
- Evidence/example: Rybbit metrics, scroll depth, outbound clicks, UTM links, Turso schema, metric snapshots.
- Personal beat: The blog is becoming a content product with feedback, not just a folder of posts.
- Mechanism: this is not full reinforcement learning; it is a practical feedback loop that reduces forgetting.

### Part 6: The Human Role Got More Important

- Point: More autonomy does not remove human judgment; it raises the price of bad judgment.
- Counterargument: This can look like tool sprawl or over-automation.
- Response: Bad delegation creates bad output faster; the human still owns priority, taste, permissions, risk, and final decisions.
- Mechanism: AI lowers execution cost, so the bottleneck shifts to routing, standards, and review.

### Part 7: The Point Is To Make Life Bigger

- Point: The purpose of leverage is not to fill every freed hour with more low-quality work.
- Evidence/example: AI can let work continue while Aaron spends time with family, exercises, reads, travels, or thinks.
- Personal beat: The California road trip is not just a productivity scene; it is the point of the productivity.
- Ending: The edge is not being the best prompter. It is building a better human-agent operating system and using the saved time to become more capable and alive.
