# I Gave Codex a Task From a Moving Tesla

## [HOOK: 00-cover-v2.png]

I was in a Tesla on the way to California.

I opened Codex on my phone and gave it a task:

Go into my Notion task tracker. Work on the blog analysis task. Use the Superpowers skill to execute it.

At the same time, Claude Code had just finished my latest development task after about 1.5 hours.

That was the moment AI stopped feeling like a tool.

It felt like an operating system.

---

## [SLIDE: From Tool To Environment — 01-metaphor-tool-to-environment-v2.png]

A tool helps with one action.

You open it, use it, close it, and move on.

[IMAGE: s01-02-tool-single-action.png]

An operating system does something different. It routes intent into execution. It manages access, keeps state, logs what happened, and decides what gets control next.

[IMAGE: s01-03-os-routes-intent.png]

That is what my AI workflow is starting to feel like. Not a better autocomplete box, but a working environment around my intent.

---

## [SLIDE: Coding Became A Run — 02-scene-coding-run-continues-v2.png]

Coding was the first door because software already has the shape agents need.

Files. Specs. Tests. Diffs. Branches. Logs. Reviewable artifacts.

[IMAGE: s02-02-repo-artifacts.png]

I no longer ask only for snippets. I give the agent a run: read the repo, understand the conventions, write a plan, implement the change, run checks, and report what changed.

[IMAGE: s02-03-review-gate.png]

The work has duration now. It moves through context, assumptions, edits, tests, and review gates.

---

## [SLIDE: Work Escaped The IDE — 03-metaphor-work-leaves-desk-v2.png]

Once this worked in coding, the pattern escaped the IDE.

Writing became a pipeline. A rough idea becomes a plan, then an outline, then an English article, a Chinese adaptation, social posts, newsletter copy, a YouTube script, and metadata.

[IMAGE: s03-02-content-pipeline.png]

The same thing started happening in image generation, Seedance video experiments, Notion task routing, scheduled jobs, analytics, and admin work.

[IMAGE: s03-03-daily-admin.png]

The important shift is not the category of work. It is that more work can become a designed workflow.

---

## [SLIDE: The Operating Loop — 05-clean-framework-intent-skill-run-review-memory-v2.png]

My current model has five parts: intent, skill, run, review, and memory.

Intent defines the outcome. Skill encodes the workflow.

[IMAGE: s04-02-intent-to-skill.png]

Run is where the agent reads, searches, edits, generates, calls tools, or asks for more context. Review is where I inspect the output and the evidence.

[IMAGE: s04-03-memory-compounds.png]

Memory is what makes the system compound. A good run should leave behind a better prompt, a better checklist, a better metric, or a better default.

---

## [SLIDE: Notion Is The Dispatch Surface — 04-light-architecture-dispatch-memory-v2.png]

Notion is useful, but it is not the whole system.

Many Notion pages are task shells. They tell the system what should happen, but they do not contain all the work memory.

[IMAGE: s05-02-notion-task-shells.png]

The real memory lives across repos, docs, specs, metrics, generated assets, and agent sessions.

[IMAGE: s05-03-distributed-memory.png]

That is why task manager is too small a phrase. The better frame is a personal operating system.

---

## [SLIDE: Self Enhancement Layer — 06-metaphor-feedback-garden-v2.png]

The next layer is self-enhancement.

For the blog, the system now treats the site like a content product, not just a folder of markdown files.

[IMAGE: s06-02-metrics-rain.png]

It tracks published content, distribution links, Rybbit metrics, scroll depth, outbound clicks, and reward scores.

[IMAGE: s06-03-next-post-grown.png]

The point is not the dashboard. The point is that the next article does not have to start from vibes alone.

---

## [SLIDE: Video Learns Too — 07-scene-creative-lab-memory-v2.png]

Video works the same way.

The Seedance lab does not just generate clips. It stores prompts, requests, outputs, critiques, and next variations.

[IMAGE: s07-02-seedance-first-frames.png]

One experiment becomes context for the next experiment.

[IMAGE: s07-03-critique-loop.png]

That is the difference between playing with a model and building a creative machine.

---

## [SLIDE: Judgment Moves Up — s08-01-operator-judgment.png]

The human role does not disappear.

It moves up a layer.

When execution becomes cheaper, the bottleneck is no longer typing every line or formatting every artifact.

[IMAGE: s08-02-permission-checkpoints.png]

The bottleneck becomes deciding what matters, setting standards, choosing what not to do, and knowing where the agent must pause.

[IMAGE: s08-03-taste-rejects-polished-wrong.png]

Bad delegation used to waste an afternoon. With agents, bad delegation can create wrong branches of work faster.

---

## [SLIDE: Life Gets Bigger — 08-closing-life-foreground-v2.png]

The point is not to work forever.

The point is to make life bigger.

If agents can research while I travel, I can be present with my family.

[IMAGE: s09-02-exercise-reading-social.png]

If background jobs can ingest metrics, I can go exercise. If workflows can structure ideas, I can spend more time reading, thinking, and talking to people.

[IMAGE: s09-03-work-background-life-foreground.png]

AI productivity is not only about doing more. It is about removing friction from the parts of life and work that were too expensive to attempt before.

---

## [SLIDE: The Durable Edge — s10-01-human-agent-os.png]

AI will not make humans obsolete.

But it can make more humans capable of operating at a higher level.

[IMAGE: s10-02-unsolved-problems.png]

The edge will not belong only to the best prompter or the person with the strongest model. Models will change. Interfaces will change.

[IMAGE: s10-03-higher-level-operator.png]

The durable edge is learning how to build a better human-agent operating system: one that can execute, report back, learn, and still leave the most human work to the human.
