# This Is My AI Workday Now

## [HOOK: 00-cover-v2.png]

This sounds like a story from 2030.

It happened on my phone, in a Tesla, on the way to California.

I opened Codex and said: go into my Notion task tracker, finish the blog analysis task, and use the Superpowers skill to execute it.

At almost the same time, Claude Code finished a development task that had been running for about an hour and a half.

No lab demo. No keynote video.

That was my workday.

And that was the moment AI stopped feeling like a tool and started feeling like an operating system.

---

## [SLIDE: From Tool To Environment — 01-metaphor-tool-to-environment-v2.png]

A tool waits for a single command.

You open it. You use it. You close it.

[IMAGE: s01-02-tool-single-action.png]

An operating system is different. It receives intent, routes work, manages context, keeps state, and decides what should happen next.

[IMAGE: s01-03-os-routes-intent.png]

That is the change I felt in the Tesla. I was not asking an app for help. I was dispatching work into an environment that could keep moving after I put the phone down.

---

## [SLIDE: Coding Became A Run — 02-scene-coding-run-continues-v2.png]

Coding was the first place this became obvious, because software already has the pieces agents need.

Files. Tests. Diffs. Logs. Pull requests. Reviewable artifacts.

[IMAGE: s02-02-repo-artifacts.png]

So the unit of work changed. I stopped asking for a snippet and started giving the agent a run: read the repo, understand the local style, write a plan, implement, test, and report back.

[IMAGE: s02-03-review-gate.png]

That is a different relationship. The agent is not answering a question. It is carrying a task through time.

---

## [SLIDE: Work Escaped The IDE — 03-metaphor-work-leaves-desk-v2.png]

Once that pattern worked in code, it escaped the IDE.

Writing became a pipeline: rough idea, plan, outline, English article, Chinese version, X post, newsletter, YouTube script, metadata.

[IMAGE: s03-02-content-pipeline.png]

Then the same shape appeared in image generation, Seedance video experiments, Notion task routing, scheduled jobs, analytics, and admin work.

[IMAGE: s03-03-daily-admin.png]

That is the first big lesson: the category matters less than the workflow shape. If the work has inputs, standards, tools, evidence, and review, it can become agent-native.

---

## [SLIDE: The Operating Loop — 05-clean-framework-intent-skill-run-review-memory-v2.png]

The model I use now has five parts: intent, skill, run, review, and memory.

Intent says what outcome I want. Skill defines how this kind of work should happen.

[IMAGE: s04-02-intent-to-skill.png]

Run is where the agent reads, searches, edits, generates, calls tools, or asks for missing context. Review is where I inspect the output and the evidence.

[IMAGE: s04-03-memory-compounds.png]

Memory is the compounding layer. A good run should leave behind a better prompt, a sharper checklist, a clearer metric, or a better default for next time.

---

## [SLIDE: Notion Is The Dispatch Surface — 04-light-architecture-dispatch-memory-v2.png]

Notion became one of my dispatch surfaces, but it is not the whole system.

Many Notion pages are task shells. They say what should happen, who owns it, and what success looks like.

[IMAGE: s05-02-notion-task-shells.png]

But the actual memory lives across repos, docs, specs, metrics, generated assets, and agent sessions.

[IMAGE: s05-03-distributed-memory.png]

That is why "task manager" feels too small. The better frame is a personal operating system, with Notion as one control surface.

---

## [SLIDE: Self Enhancement Layer — 06-metaphor-feedback-garden-v2.png]

The next layer is self-enhancement.

For the blog, the system now treats the site like a content product, not just a folder of markdown files.

[IMAGE: s06-02-metrics-rain.png]

It tracks published content, distribution links, Rybbit metrics, scroll depth, outbound clicks, reward scores, and lessons from previous posts.

[IMAGE: s06-03-next-post-grown.png]

The point is not to worship a dashboard. The point is that the next article starts with memory instead of vibes alone.

---

## [SLIDE: Video Learns Too — 07-scene-creative-lab-memory-v2.png]

Video is going through the same transition.

The Seedance lab is not only generating clips. It stores prompts, requests, outputs, critiques, and next variations.

[IMAGE: s07-02-seedance-first-frames.png]

One experiment becomes context for the next experiment.

[IMAGE: s07-03-critique-loop.png]

That is the difference between playing with a model and building a creative machine. The output matters, but the learning loop matters more.

---

## [SLIDE: Judgment Moves Up — s08-01-operator-judgment.png]

This does not make the human role smaller.

It moves the human role up a layer.

When execution gets cheaper, the bottleneck is no longer typing every line or formatting every artifact.

[IMAGE: s08-02-permission-checkpoints.png]

The bottleneck becomes judgment: deciding what matters, setting standards, choosing what not to do, and knowing where the agent must pause.

[IMAGE: s08-03-taste-rejects-polished-wrong.png]

There is a risk here. Bad delegation used to waste an afternoon. With agents, bad delegation can create polished wrong work much faster.

---

## [SLIDE: Life Gets Bigger — 08-closing-life-foreground-v2.png]

The goal is not to work forever.

The goal is to make life bigger.

If agents can research while I travel, I can be present with my family.

[IMAGE: s09-02-exercise-reading-social.png]

If background jobs can ingest metrics, I can go exercise. If workflows can structure ideas, I can spend more time reading, thinking, and talking to people.

[IMAGE: s09-03-work-background-life-foreground.png]

AI productivity is not only about doing more. It is about removing friction from the work and life that used to be too expensive to attempt.

---

## [SLIDE: The Durable Edge — s10-01-human-agent-os.png]

AI will not make humans obsolete.

But it can help more people operate at a higher level.

[IMAGE: s10-02-unsolved-problems.png]

The edge will not belong only to the person with the strongest model. Models will change. Interfaces will change. Prices will change.

[IMAGE: s10-03-higher-level-operator.png]

The durable edge is learning how to build a better human-agent operating system: one that can execute, report back, learn, and still leave the most human work to the human.
