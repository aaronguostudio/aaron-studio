# June 2026 Development Summary — Viewpoint Vista Tooling

**Consultant:** Aaron Guo (`aaronguo`) · **Period:** June 2026 · **Products:** vgpt Excel add-in + vgpt3 MCP/knowledge backend

> Prepared as an invoice-ready summary. All figures are approximate/indicative and grounded in the June git history of both repositories. See the Attribution note for items excluded from billing.

---

## 1. Executive summary

In June the vgpt Excel add-in gained several capabilities it did not have on June 1. Users can now turn plain-English questions into real **Crystal Reports `.rpt` files** rendered directly against their own live Vista database — starting with a production AP Aging template and growing into a full **custom report builder** (arbitrary columns, grouping, subtotals/grand totals, and formatting) that can be requested and downloaded straight from Excel chat. The add-in can also pull **live Vista data into a workbook as a refreshable Power Query** with no database credentials or VPN ever touching the spreadsheet, via either an AI-written-and-validated query or a user's own verbatim SQL run exactly as given. The assistant's **VBA authoring became trustworthy** — a structural validator, a real full-project compile check, a SQL-in-VBA schema validator that blocks hallucinated tables/columns, and a vetted "golden pattern" for importing spreadsheet data into Vista custom tables. On top of this, the chat experience was made **persistent and first-class** (survives workbook reopen, pops out into its own window that keeps working during cell editing, and was restyled to match the web product), the **web chat model was upgraded to Claude Opus 4.8** with visible extended reasoning, and the **core Excel layer was hardened** (worksheet switching, workbook awareness, and automatic recovery from out-of-memory large writes). This shipped as roughly **17 add-in MSI releases (v1.7.5.0 → v1.7.19.0)**, backed by a new automated quality-assurance/benchmark system and prompt-injection hardening.
