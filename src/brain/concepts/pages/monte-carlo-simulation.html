Monte Carlo
Simulation

未来不是一条线，
而是一片路径。

它不告诉你哪条路径会发生；它让你看见，在一组明确假设下，可能结果有多宽、失败藏在哪里。

01 · The problem before the definition

一条平均线，会把最重要的事藏起来。

“未来 30 年每年回报 6%”很容易算，却没有告诉你中途会不会先跌 25%、资金何时可能耗尽，或坏结果到底是偶发还是结构性脆弱。

One-sentence mental model

把不确定输入反复抽样成许多条可能路径，再用结果分布回答范围、尾部、失败概率与敏感因素。

—

单一路径

户外婚礼只看“平均 22°C”，然后默认那天不需要备用方案。数字很干净，决策信息却很少。

∿

Monte Carlo Simulation

把气温、降雨和风速按合理关系重播一万次：不是预测真实那天，而是决定帐篷值不值得准备。

04 · Under the hood

Monte Carlo 不是“多做几个情景”，而是一条完整的不确定性管道。

最难的通常不是随机数，而是定义哪些世界可信、变量如何一起移动，以及什么结果才算失败。

01

Frame

先定义决策、时间尺度、成功阈值和输出单位。

02

Model

写出结果如何由现金流、回报或经营驱动因素产生。

03

Sample

从输入分布联合抽样，保留相关性与因果约束。

04

Repeat

让每个世界走完路径，再重复成百上千次。

05

Challenge

读分布、尾部和失败路径，再做敏感度与压力测试。

Model · Y = f(X₁, X₂, …, Xₖ)
One run · draw jointly: X⁽ⁱ⁾ → compute Y⁽ⁱ⁾
Repeat · i = 1 … N → empirical distribution of Y

关键边界：增加 N 只会降低 Monte Carlo sampling error（蒙特卡洛抽样误差）。如果输入分布、相关性、现金流机制或目标函数错了，十万次运行只是把同一个错误算得更稳定。

05 · Same engine, different question

长期资金规划和企业价值分析，用的是同一台机器。

差别不在“有没有随机数”，而在输入、业务模型、输出与最终决策问题。切换两种镜头看同一套机制。

06 · Concept neighborhood

最容易误会的，是那些会一起出现的方法。

它们不是 Monte Carlo 的不同名字，而是输入机制、基础模型、补充证据或输出指标。

Scenario Analysis NARRATIVE METHOD

情景分析

用少量一致、可解释的未来讲清因果；Monte Carlo 则系统抽取大量可能组合。

Stress Testing COMPLEMENTARY EVIDENCE

压力测试

强迫模型进入指定坏世界。它覆盖模拟分布可能几乎从不生成的危机与制度切换。

Sensitivity Analysis DIAGNOSTIC METHOD

敏感度分析

解释哪个输入最推动结果；Monte Carlo 负责传播联合不确定性，两者常一起使用。

Bootstrap SAMPLING MECHANISM

自助抽样

从经验数据重复抽样；它可以成为 Monte Carlo 的输入生成方法，区块抽样还能保留部分时间依赖。

DCF VALUATION MODEL

Discounted Cash Flow · 贴现现金流

DCF 是计算价值的 f(·)；Monte Carlo 反复给它不同输入，输出内在价值分布。

VaR OUTPUT METRIC

Value at Risk · 在险价值

可以从模拟结果读取的损失分位数；它不是模拟方法，也不告诉你超过阈值以后会有多糟。

Sequence Risk APPLICATION RISK

Sequence-of-Returns Risk · 收益顺序风险

提款或现金流让年度回报顺序改变结局；它是路径模拟在长期资金规划中最直观的应用之一。

Model Risk PARENT RISK

模型风险

模型结构、数据、参数和使用方式本身造成的风险。Monte Carlo 能展示模型内不确定性，不能消除模型风险。

07 · Failure modes and judgment

模拟看起来越精密，越要问它在哪一层可能错。

把不确定性分三层：随机抽样的噪音、模型本身的错误，以及把模型输出翻译成行动时的判断风险。

Sampling error

同一模型换一个随机种子，分位数略有变化。增加运行次数可以缩小这层误差。

More runs can help

Model risk

厚尾、相关性飙升、税、流动性、行为反应或制度切换被漏掉。更多运行次数无法修复。

Challenge assumptions

Decision risk

把 84.7% 当成确定性，或为了得到漂亮数字反向调假设。需要治理、解释与行动规则。

Use judgment

选择一个情景，检查你是在修复抽样误差、模型风险，还是决策解释。

08 · Pocket memory

五件事，带走就够。

  1. 它不是一条预测线，而是一组以假设为条件的可能路径。
  2. 机制是：定义输入与关系 → 联合抽样 → 跑完整模型 → 重复 → 看分布。
  3. 最有用的输出不是平均值，而是范围、尾部、失败路径与敏感因素。
  4. 更多运行次数减少抽样噪音，不减少模型风险。
  5. 长期资金规划看目标能否撑住；企业价值分析看价值如何分布——机器相同，问题不同。
