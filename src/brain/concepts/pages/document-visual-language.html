01 · The problem

“更专业一点”不是设计规格。

Artificial Intelligence（AI，人工智能）让 Portable Document Format（PDF，便携式文档格式）的生成成本迅速下降。但如果团队只能用“高级、温暖、科技感”评审结果，生成越快，抽卡和返工也会越快。

One-sentence definition

文档视觉语言把希望读者怎样感受、怎样阅读、怎样行动，翻译成可重复的版式、字体、颜色、密度与组件规则。

03 · Five-layer stack

名字只在第四层。前三层解释选择，最后一层负责落地。

同一套颜色不能让研究报告、品牌杂志和操作手册自动成为同一种设计。视觉语言从读者任务开始，一层层走到可实现的参数与组件。

01

Document GenreFunction contract

报告、长文、手册、提案或专题。它定义读者主要是在阅读、扫描、执行还是决策。

02

Design LineageHistorical grammar

Swiss、New Typography 或古典书籍传统。它提供已经形成边界的构图与层级关系。

03

Style ArchetypeWorking vocabulary

Organic Humanist、Technical Functional、Institutional Modern 等当代工作型原型。

04

Theme PresetNamed instance

Alder、Granite 等可记忆的具体实现。名字属于产品，不自动拥有行业通用含义。

05

Tokens + ComponentsExecutable rules

字体、字号、行高、颜色、间距、网格，以及标题、表格、引语、警告与页脚的行为。

Visual language = intent + genre + lineage + archetype + theme + tokens + component rules + accessibility constraints
04 · Style dictionary

先标明“这是什么类型的名字”，再谈它长什么样。

历史运动、文档类型、工作型原型和跨领域类比不能装成同一级别的标准。好的词典会把身份与边界一起保存。

Convention

Classical Bookish

安静的衬线正文、稳定比例与宽页边距。适合连续阅读；数据密集时会显得迟缓。

Historical movement

Swiss / International

无衬线、网格、非对称、摄影与明确层级。适合信息报告；失败时只剩企业冷感。

Historical movement

New Typography

文字块、图像、标准格式和强烈对比。适合海报与专题进入页，不适合每页都持续喊叫。

Genre + method

Editorial

展示字体、图片、分栏、引语与节奏变化。适合故事；缺少规则时会变成随机构图。

Working archetype

Technical Functional

模块网格、编号、表格、细线与较高密度。适合规范和审计；拥挤不等于专业。

Working archetype

Organic Humanist

暖中性色、人文字体、柔和边界和舒展节奏。适合教育与生活方式；低对比会削弱可信度。

Working archetype

Institutional Modern

冷灰、严格网格、克制无衬线和稳定重复。适合治理和金融；容易正确却不可记忆。

Working archetype

Luxury Editorial

高对比展示衬线、大量留白、细线与少量颜色。适合品牌展示；小字浅灰会让“高级”变难读。

Transferred analogy

Brutalist / Raw

巨大字体、硬边框、裸露结构和强反差。适合文化实验；阅读困难不是态度本身。

06 · Failure modes

风格失败，很少只是“颜色选错了”。

真正的失败通常发生在层级错位：名字没有规格、风格压过任务、参数没有关系，或者视觉漂亮却破坏阅读与语义。

Poetry without specification

名字有意象，没有原型、坐标和参数。每次实现都重新猜一次。

Style soup

Swiss、奢华、温暖、粗野、极简、高密度全部都要，却没有优先级。

Fashion before function

为了喜欢的视觉，把操作手册做成杂志，把数据证据藏在“极简”背后。

Token theater

有数百个 Design Token，却说不清读者任务和组件怎样协同。

Alias drift

不同团队都说 Granite，一个指金融报告，另一个指深色技术手册。

Accessibility laundering

用“高级”解释低对比，用“实验”解释错误阅读顺序。风格不能替代语义结构与可访问性。

07 · Concept neighborhood

最容易混淆的，是同一套页面上的不同系统。

把功能、感知、结构、实现与品牌分开以后，团队才知道一次反馈应该改哪一层。

Visual Style PERCEPTION

读者感受到的稳定性格。视觉语言还包含任务、结构与实现。

Document Genre FUNCTION

读者来完成什么工作。报告与杂志可以共享颜色，却不能共享阅读结构。

Layout System STRUCTURE

网格、分栏、页边距和基线，是骨架，不单独决定语气。

Typography System SUBSYSTEM

字体角色、字号层级、行长与行高。很显眼，但不是全部。

Design Tokens IMPLEMENTATION DATA

保存颜色、尺寸、字体和间距的命名值，不负责判断它是否适合任务。

Brand Identity CROSS-MEDIA SYSTEM

跨标志、语气、图像和接触点的身份；文档视觉语言只是其中一种表达。

Template CONTENT SKELETON

保存重复页面结构。一个模板可以换主题，一个主题也可以服务多个模板。

Progressive Disclosure INFORMATION STRATEGY

决定复杂度何时出现；视觉语言决定出现以后怎样建立层级与密度。

一句辨别：Theme 是已经命名的一套实现；Style 是它表现出的性格；Layout 是空间结构；Typography 是文字子系统；Design Token 是可交换的实现值。它们互相依赖，但不是同义词。
08 · Pocket memory

最后，只记住这五件事。

  • 01主题名让人记住，视觉语言让团队做对。
  • 02先确定文档类型，再选择设计谱系和工作型原型，最后才给主题命名。
  • 03用 Warm–Cool、Soft–Hard、Sparse–Dense、Restrained–Expressive 说明方向和强度。
  • 04Design Token 保存数值，组件规则保存关系；两者都不能代替沟通意图。
  • 05风格不能修复错误内容，也不能替代可读性、语义结构与可访问性。
